## 2. 原始已有转换命令

### 2.1 原始数据同步

进入脚本目录：

```bash
cd ~/pika_ros/src/data_tools/scripts
```

原始双臂遥操同步命令：

```bash
python3 data_sync.py --type multi_pika_teleop --datasetDir $HOME/agilex/data/
```

原始单臂遥操同步命令：

```bash
python3 data_sync.py --type single_pika_teleop --datasetDir $HOME/agilex/data/
```

同步完成后，每个传感器目录下会生成 `sync.txt`。

### 2.2 原始 HDF5 转换

如果不使用点云：

```bash
cd ~/pika_ros/src/data_tools/scripts
python3 data_to_hdf5.py --type multi_pika_teleop --datasetDir $HOME/agilex/data/ --useCameraPointCloud ""
```

如果需要导出非索引型 HDF5：

```bash
cd ~/pika_ros/src/data_tools/scripts
python3 data_to_hdf5.py \
  --type multi_pika_teleop \
  --datasetDir $HOME/agilex/data/ \
  --useIndex "" \
  --useCameraPointCloud "" \
  --datasetTargetDir $HOME/agilex/hdf5_export/
```

## 3. 当前新结构对应的计划新增命令

下面这些命令是后续准备固定下来的新命令名，用于你的新数据结构：

- 无 fisheye
- 额外包含 head cam
- 在转换层把 `myD435` 映射为 `headCam`

建议新类型名固定为：

```bash
multi_pika_teleop_headcam_no_fisheye
```

### 3.1 新的数据同步命令

建议新增脚本后使用：

```bash
cd ~/pika_ros/scripts/custom_data_tools
python3 data_sync_headcam_no_fisheye.py \
  --datasetDir $HOME/agilex/pour \
  --type multi_pika_teleop_headcam_no_fisheye
```

如果处理其他任务目录，例如 `pnp_star_pear`：

```bash
cd ~/pika_ros/scripts/custom_data_tools
python3 data_sync_headcam_no_fisheye.py \
  --datasetDir $HOME/agilex/pnp_star_pear \
  --type multi_pika_teleop_headcam_no_fisheye
```

### 3.2 新的 HDF5 转换命令

默认索引型 HDF5：

```bash
cd ~/pika_ros/scripts/custom_data_tools
python3 data_to_hdf5_headcam_no_fisheye.py \
  --datasetDir $HOME/agilex/pour \
  --type multi_pika_teleop_headcam_no_fisheye
```

非索引型 HDF5：

```bash
cd ~/pika_ros/scripts/custom_data_tools
python3 data_to_hdf5_headcam_no_fisheye.py \
  --datasetDir $HOME/agilex/pour \
  --type multi_pika_teleop_headcam_no_fisheye \
  --useIndex "" \
  --datasetTargetDir $HOME/agilex/hdf5_export/pour
```

### 3.3 新命令与原命令的区别

原命令：

- 默认认为 fisheye 存在
- 不认识 `myD435 -> headCam` 的映射
- 默认面向 `~/agilex/data`

新命令：

- 专门适配无 fisheye 结构
- 专门适配任务目录，例如 `~/agilex/pour`
- 在输出 key 层把 `myD435` 映射成 `headCam`

特别注明：

- `camera/color/pikaGripperDepthCamera_l` 是左手腕 RGB
- `camera/color/pikaGripperDepthCamera_r` 是右手腕 RGB
- `camera/color/myD435` 是头部 RGB
- 如果进一步处理为 RobotWin 风格数据：
  - 左手腕 RGB -> `cam_left_wrist`
  - 右手腕 RGB -> `cam_right_wrist`
  - 头部 RGB -> `cam_high`

## 4. 转换后如何查看 key

推荐用 Python + `h5py` 查看，最稳妥。

### 4.1 查看顶层 key

```bash
python3 - <<'PY'
import h5py
path = "/home/piper/agilex/pour/episode0/data.hdf5"
with h5py.File(path, "r") as f:
    print(list(f.keys()))
PY
```

### 4.2 递归查看全部 key

```bash
python3 - <<'PY'
import h5py
path = "/home/piper/agilex/pour/episode0/data.hdf5"
def show(name, obj):
    print(name)
with h5py.File(path, "r") as f:
    f.visititems(show)
PY
```

### 4.3 查看某个 key 的形状和类型

```bash
python3 - <<'PY'
import h5py
path = "/home/piper/agilex/pour/episode0/data.hdf5"
with h5py.File(path, "r") as f:
    d = f["camera/color/headCam"]

PY
```

如果系统里安装了 HDF5 命令行工具，也可以用：

```bash

```

## 5. 如何校验转换是否正确

建议按下面顺序检查。

### 5.1 先检查原始目录是否完整

例如检查 `episode0`：

```bash
find /home/piper/agilex/pour/episode0/camera -maxdepth 3 | sort
```

应当至少看到：

- `camera/color/pikaGripperDepthCamera_l`
- `camera/color/pikaGripperDepthCamera_r`
- `camera/color/myD435`
- `camera/depth/pikaGripperDepthCamera_l`
- `camera/depth/pikaGripperDepthCamera_r`

### 5.2 检查 `sync.txt` 是否生成

```bash
find /home/piper/agilex/pour/episode0 -name sync.txt | sort
```

应当在每个参与同步的目录下看到 `sync.txt`。

### 5.3 检查各 `sync.txt` 行数是否一致

```bash
wc -l \
  /home/piper/agilex/pour/episode0/camera/color/pikaGripperDepthCamera_l/sync.txt \
  /home/piper/agilex/pour/episode0/camera/color/pikaGripperDepthCamera_r/sync.txt \
  /home/piper/agilex/pour/episode0/camera/color/myD435/sync.txt \
  /home/piper/agilex/pour/episode0/camera/depth/pikaGripperDepthCamera_l/sync.txt \
  /home/piper/agilex/pour/episode0/camera/depth/pikaGripperDepthCamera_r/sync.txt
```

正常情况下，这几路同步后的行数应一致，或者只允许极少量可解释的裁剪差异。

### 5.4 检查 HDF5 是否生成

```bash
ls -lh /home/piper/agilex/pour/episode0/data.hdf5
```

### 5.5 检查关键 key 是否存在

原始方案检查：

- `camera/color/pikaGripperDepthCamera_l`
- `camera/color/pikaGripperDepthCamera_r`
- `camera/color/pikaGripperFisheyeCamera_l`
- `camera/color/pikaGripperFisheyeCamera_r`

新方案检查：

- `camera/color/pikaGripperDepthCamera_l`
- `camera/color/pikaGripperDepthCamera_r`
- `camera/color/headCam`
- `camera/depth/pikaGripperDepthCamera_l`
- `camera/depth/pikaGripperDepthCamera_r`

### 5.6 检查 `size` 与时间戳长度

```bash
python3 - <<'PY'
import h5py
path = "/home/piper/agilex/pour/episode0/data.hdf5"
with h5py.File(path, "r") as f:
    print("size =", f["size"][()])
    print("timestamp_len =", len(f["timestamp"]))
PY
```

这两个值应当一致或逻辑一致。

### 5.7 抽查几条索引路径是否真实存在

如果是索引型 HDF5：

```bash
python3 - <<'PY'
import h5py
from pathlib import Path
path = Path("/home/piper/agilex/pour/episode0/data.hdf5")
root = path.parent
with h5py.File(path, "r") as f:
    for key in [
        "camera/color/pikaGripperDepthCamera_l",
        "camera/color/pikaGripperDepthCamera_r",
        "camera/color/headCam",
    ]:
        value = f[key][0]
        if isinstance(value, bytes):
            value = value.decode()
        full = root / value
        print(key, "->", full, "exists=", full.exists())
PY
```

应当全部存在。

## 6. 新方案转换正确后的预期

如果新方案转换正确，应当满足：

- fisheye 不再出现在输出 key 中
- `myD435` 不再作为对外 key 名出现
- `headCam` 成为新的头部相机字段名
- 各传感器 `sync.txt` 行数基本一致
- `size`、`timestamp`、图像索引长度互相一致
- 随机抽查的图像路径真实存在

## 7. 当前原始采集目录直接处理为 RobotWin 风格数据

如果不先转 `data.hdf5`，而是直接把当前原始 episode 目录处理成 RobotWin 风格数据，建议使用新的独立脚本：

```bash
python /home/piper/pika_ros/scripts/process_data_robotwin_headcam.py \
  /home/piper/agilex/pnp_star_pear \
  "Pick up the starfruit and the pear, then place them onto the blue plate." \
  50
```

指定输出目录的完整例子：

```bash
python /home/piper/pika_ros/scripts/process_data_robotwin_headcam.py \
  /home/piper/agilex/pnp_star_pear \
  "Pick up the starfruit and the pear, then place them onto the blue plate." \
  50 \
  --output-dir /home/piper/agilex/processed_robotwin/pnp_star_pear-50
```

该脚本的固定约定：

- 输入是当前原始采集目录 `~/agilex/<task>/episode*`
- `myD435` 固定解释为头部相机 `headCam`
- 三路 RGB 固定对应：
  - 左手腕
  - 右手腕
  - 头部相机
- 输出为 RobotWin 风格目录：
  - `processed_data/<task>-<N>/episode_i/instructions.json`
  - `processed_data/<task>-<N>/episode_i/episode_i.hdf5`

输出 HDF5 的关键字段：

- `action`
- `observations/state`
- `observations/images/cam_high`
- `observations/images/cam_left_wrist`
- `observations/images/cam_right_wrist`

特别说明：

- 当前 raw 数据没有单独的 action topic
- 新脚本默认使用“下一时刻的 puppet 状态”作为 action
- 当前状态向量和动作向量均为 14 维：
  - `left_pos(3)`
  - `left_euler(3)`
  - `left_gripper(1)`
  - `right_pos(3)`
  - `right_euler(3)`
  - `right_gripper(1)`

当前这个脚本明确未使用：

- 双路 depth 图像
- master 端末端位姿
- jointState
- sensor 夹爪编码器
- localization
- episode 内原始 `instructions.json`

当前这个脚本的筛选规则：

- 结构异常直接跳过并记日志：
  - 缺少 `cam_high`
  - 缺少 `cam_left_wrist`
  - 缺少 `cam_right_wrist`
  - 缺少组成 state/action 所需的 `puppetLeft` / `puppetRight` / `pikaGripper_l` / `pikaGripper_r`
- RGB 最低 hz 小于 `8`：
  - 直接跳过
  - 红色终端输出
  - 记入同一个审计日志
- RGB 最低 hz 在 `[8, 10)`：
  - 黄色终端输出
  - 保留处理结果
  - 同时记入审计日志，供后续人工审核

审计日志路径：

- `processed_output_dir/processing_audit.jsonl`

## 8. 处理后内容检查脚本

处理完成后，建议使用独立检查脚本：

```bash
python /home/piper/pika_ros/scripts/check_processed_robotwin_headcam.py \
  /home/piper/agilex/processed_robotwin/pnp_star_pear-50 \
  --episode 0
```

它会检查：

- `instructions.json` 是否存在
- `episode_0.hdf5` 是否存在
- `action` 和 `observations/state` 长度是否一致
- `cam_high` / `cam_left_wrist` / `cam_right_wrist` 是否存在
- 图像字节是否能成功解码

如果你想快速看多个 episode，可以先手动抽查：

```bash
for i in 0 1 2; do

    /home/piper/agilex/processed_robotwin/pnp_star_pear-50 \

```
