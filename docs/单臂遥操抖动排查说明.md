# 单臂遥操抖动排查说明

这份说明的目标很简单：

- 先把抖动拆成 3 段
- 再决定问题是在 `sense 定位`、`teleop 映射/IK`，还是 `机械臂执行`

建议同时使用两套工具：

- 在线看：`scripts/monitor_single_piper_jitter_tmux.bash`
- 离线录：`scripts/record_single_piper_debug.bash`

## 一、最直接的启动方式

如果你当前 shell 是 `zsh`：

```bash
PIKA_ROS_SETUP=/home/piper/pika_ros/install/setup.zsh bash /home/piper/pika_ros/scripts/monitor_single_piper_jitter_tmux.bash
tmux attach -t piper-jitter
```

会打开 5 个窗口：

- `pose_probe`
- `gripper_probe`
- `hz`
- `status`
- `record`

## 二、先理解 3 段链路

### 1. 输入侧

这是 `sense` 和定位系统出来的数据。

主要看：

- `/pika_pose`
- `/pika_localization_status`
- `/sensor/gripper/joint_state`

### 2. 目标侧

这是 teleop 和 IK 希望机械臂去的位置。

主要看：

- `/piper_IK/ctrl_end_pose`
- `/joint_states_gripper`

### 3. 执行侧

这是机械臂真实反馈回来的结果。

主要看：

- `/joint_states_single`
- `/piper_FK/urdf_end_pose_orient`

## 三、在线怎么看

### 1. `pose_probe`

这是最有用的窗口。

每秒会打印：

- `PIKA`
  - 遥操输入位姿
- `CTRL`
  - teleop 发给 IK 的目标末端位姿
- `FK`
  - 机械臂实际末端位姿

还会打印每条 topic 的：

- `age_ms`
  - 最新消息年龄
- `hz`
  - 实际接收频率
- `jitter_ms`
  - 消息间隔抖动
- `step_poserpy`
  - 相邻两帧变化幅度

### 2. `gripper_probe`

每秒打印夹爪链路：

- `SENSE`
  - 遥操端夹爪开口
- `MERGED`
  - 合并后的夹爪控制开口
- `JOINT7`
  - 真机反馈的第 7 轴开口

### 3. `hz`

这个窗口只看频率，适合快速发现某个 topic 掉频或者断流。

### 4. `status`

重点看：

- `localization accurate`
- `teleop (fail, quit)`
- `arm over_limit`

### 5. `record`

这里只给你一条录制命令。

比如录 60 秒：

```bash
RECORD_SECONDS=60 bash /home/piper/pika_ros/scripts/record_single_piper_debug.bash
```

## 四、怎么判断抖动来自哪里

### 情况 A：`PIKA` 自己就在跳

表现：

- `pose_probe` 里 `PIKA step_poserpy` 很大
- `/pika_pose` 的 `jitter_ms` 很大
- `CTRL` 也跟着跳

结论：

- 更像是 `sense` 定位端的问题

优先检查：

- 基站摆放
- 标定是否失效
- USB 重插后设备枚举是否变化
- `s2` 里相机/定位报错

### 情况 B：`PIKA` 稳，但 `CTRL` 在跳

表现：

- `PIKA step_poserpy` 不大
- `CTRL step_poserpy` 明显更大
- 或者 `CTRL-FK` 差值突然大跳

结论：

- 更像是 teleop 映射、触发重新锁存、姿态映射或者 IK 输出不稳定

优先检查：

- 是否误触发过 teleop
- `/teleop_status` 是否有异常切换
- `/arm_control_status` 是否反复 `over_limit`

### 情况 C：`CTRL` 稳，但 `FK` 在抖或者跟不上

表现：

- `CTRL` 变化平滑
- `FK step_poserpy` 抖动大，或者明显滞后
- `/joint_states_single` 抖

结论：

- 更像是执行侧问题

优先检查：

- 机械臂本体是否软
- CAN 是否稳定
- 电机使能状态
- 机械臂是否真正执行了关节命令

### 情况 D：夹爪稳、位置轴抖

结论通常是：

- 夹爪链路正常
- 位置轴链路异常

因为夹爪控制和 6 轴位置控制不是同一条执行路径。

## 五、离线录制怎么用

录制命令：

```bash
RECORD_SECONDS=60 PIKA_ROS_SETUP=/home/piper/pika_ros/install/setup.zsh bash /home/piper/pika_ros/scripts/record_single_piper_debug.bash
```

默认会把数据放到：

```text
/home/piper/pika_ros/logs/jitter_runs/<时间戳>/
```

里面包括：

- `single_piper_debug.bag`
- `rosnode_list.txt`
- `rostopic_list.txt`
- 每个关键 topic 的 `rostopic info`
- `topic_hz.txt`

## 六、一个建议的排查顺序

1. 先打开 `piper-jitter`
2. 观察 20 到 30 秒
3. 如果看见抖动，再录 60 秒 rosbag
4. 先判断是 `PIKA` 抖、`CTRL` 抖，还是 `FK` 抖
5. 再决定是去查 `s2`、`s3`，还是机械臂执行侧

## 七、最直观的经验判断

如果你肉眼感觉：

- 遥操设备本身很稳
- 但机械臂末端一顿一顿

最值得先看的是：

- `CTRL` 是否平滑
- `FK` 是否跟不上

如果：

- `CTRL` 很平滑
- `FK` 很抖

那就不要继续纠结 `sense` 侧了，优先查执行链路。
