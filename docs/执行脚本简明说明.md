# 执行脚本简明说明

本文档只说明一件事：单臂 Piper + sensor/gripper 遥操时，每个终端该怎么执行，避免 `yaml`、`cv_bridge`、`setup.bash` 之类的环境问题。

## 一、为什么你现在会报 `yaml` 缺失

当前 `s2` 里报错是：

```text
ModuleNotFoundError: No module named 'yaml'
```

报错进程是：

- `sensor_camera_fisheye`
- `gripper_camera_fisheye`
- 对应脚本：`usb_camera.py`

这说明鱼眼相机节点启动时，用到了错误的 Python 环境。

已经确认：

- `/usr/bin/python3` 是有 `yaml` 模块的
- `~/miniconda3/bin/python3` 没有 `yaml` 模块

所以这不是 ROS 本身坏了，而是启动时混进了 conda 的 Python。

## 二、你前面的命令哪里不对

你写的这两种里，第一种有明显问题：

```bash
conda deactivate
source ~/pika_ros/install/setup.bash
cd ~/pika_ros/scripts && bash start_sensor_gripper.bash
```

如果你当前 shell 是 `zsh`，这里不应该直接 `source setup.bash`。

原因：

- `setup.bash` 是给 `bash` 用的
- `setup.zsh` 才是给 `zsh` 用的

所以：

- 在 `zsh` 终端里，不能先手动 `source ~/pika_ros/install/setup.bash`
- 在 `zsh` 终端里，如果只是想运行 bash 脚本，直接 `bash start_sensor_gripper.bash` 就行

第二种比第一种更接近正确：

```bash
conda deactivate
source ~/pika_ros/install/setup.zsh
cd ~/pika_ros/scripts && bash start_sensor_gripper.bash
```

但它仍然可能不够稳，因为 conda 虽然 `deactivate` 了，环境变量和 PATH 顺序有时还会残留影响。

## 三、推荐的终端 2 启动方法

如果你当前终端是 `zsh`，推荐直接这样：

```bash
conda deactivate
export PATH=/usr/bin:/bin:/usr/sbin:/sbin:$PATH
unset PYTHONHOME
unset PYTHONPATH
source /opt/ros/noetic/setup.zsh
source ~/pika_ros/install/setup.zsh
cd ~/pika_ros/scripts
bash start_sensor_gripper.bash
```

这套命令的目的很简单：

- 强制优先用系统 Python
- 去掉 conda 对 ROS Python 节点的污染
- 再加载 ROS 和工作区环境

## 四、推荐的终端 3 启动方法

终端 3 用来启动机械臂侧 teleop，建议这样：

```bash
conda activate pika
source ~/pika_ros/install/setup.zsh
roslaunch pika_remote_piper teleop_rand_single_piper.launch
```

注意：

- 如果当前 shell 是 `zsh`，就用 `setup.zsh`
- 不要在 `zsh` 里手动 `source setup.bash`

## 六、如何判断终端 2 的环境对了

如果环境正确：

- 不应该再看到 `ModuleNotFoundError: No module named 'yaml'`
- `sensor_camera_fisheye`
- `gripper_camera_fisheye`
  不应该反复重启

如果还报 `yaml`：

- 说明当前启动仍然走到了 conda 的 Python
- 优先检查是不是还在旧 tmux 会话里复用老环境

## 八、为什么 `piper-low` 也会失败

低频监控 tmux `piper-low` 失败的原因，和终端 2 的 `yaml` 问题本质相同：

- tmux pane 里看到的是 `(base)`
- 也就是 conda base 还在
- 这时 `python3` 实际指向 `~/miniconda3/bin/python3`
- 它没有 `yaml`
- 所以 `lowfreq_single_piper_monitor.py` 一导入 `rospy` 就失败

典型报错是：

```text
ModuleNotFoundError: No module named 'yaml'
```

修正方式：

- 监控脚本里强制使用 `/usr/bin/python3`
- 启动前强制：
  - `export PATH=/usr/bin:/bin:/usr/sbin:/sbin:$PATH`
  - `unset PYTHONHOME`
  - `unset PYTHONPATH`

修正后重新启动：

```bash
tmux kill-session -t piper-low
PIKA_ROS_SETUP=/home/piper/pika_ros/install/setup.zsh bash /home/piper/pika_ros/scripts/monitor_single_piper_lowfreq_tmux.bash
tmux attach -t piper-low
```

## 七、最简单的原则

记住这 3 条就够了：

1. `zsh` 里用 `setup.zsh`
2. 跑传感器脚本前先 `conda deactivate`
3. 跑传感器脚本前优先切回系统 Python 路径
